# Basho DB マッチングサイト — チーム編成

## プロジェクト概要

場所性データベース（102施設）を基盤に、飲食店・宿泊施設の「場所性マッチングサイト」を構築する。

## チーム構成

### コアチーム

| 役割 | 担当エージェント | モデル | 責任範囲 |
|------|------------------|--------|----------|
| **プロダクトマネジャー** | process-orchestrator | Opus | 全体統制、ステージ管理、品質ゲート |
| **仕様策定** | spec-writer | Opus | 要件定義、UI仕様、ユーザーストーリー |
| **リサーチ** | researcher | Haiku | ベンチマーク調査、技術調査、データソース調査 |
| **設計** | Plan agent | Sonnet | アーキテクチャ設計、DB設計、API設計 |
| **フロントエンド実装** | engineer skill | Sonnet | Next.js UI構築、コンポーネント開発 |
| **場所性エージェント** | basho_agent.py | Claude API | 施設の8軸場所性評価 |

### QA・レビューチーム

| 役割 | 担当エージェント | モデル | 責任範囲 |
|------|------------------|--------|----------|
| **QAリード** | qa-lead | Opus | テスト統制、品質スコア |
| **UXリード** | ux-lead | Opus | UXレビュー統制、デザイン品質 |
| **セキュリティ** | security-auditor | Sonnet | OWASP準拠チェック |
| **最終ゲート** | sentinel | Opus | デプロイ前最終承認 |

### デプロイ・運用チーム

| 役割 | 担当エージェント | モデル | 責任範囲 |
|------|------------------|--------|----------|
| **デプロイ** | deploy-manager | Sonnet | GitHub push、Vercelデプロイ、Notion記録 |
| **モニタリング** | monitor-observer | Haiku | デプロイ後観測 |

## 技術スタック（ベンチマーク調査に基づく推奨）

| レイヤー | 技術 | 選定理由 |
|----------|------|----------|
| フレームワーク | Next.js (App Router) | SSG/SSR対応、画像最適化、日本での実績 |
| スタイリング | Tailwind CSS | 迅速なUI構築、レスポンシブ対応 |
| データベース | SQLite (basho.db) | 既存DB活用、Phase 1で102件構築済み |
| CMS | ローカルJSON/SQLite → Phase 2でヘッドレスCMS検討 | 段階的移行 |
| 画像 | next/image + Unsplash/自前写真 | 最適化自動、WebP/AVIF対応 |
| ホスティング | Vercel | Next.js最適、既存運用実績 |
| AI評価 | Claude API (basho_agent.py) | 場所性8軸評価 |

## 開発フェーズ

### Phase 2a: マッチングサイトMVP
1. トップページ（ビジュアル重視、キュレーション型）
2. 施設一覧ページ（カード型 + フィルタ）
3. 施設詳細ページ（写真ギャラリー + 場所性レーダーチャート + ストーリー）
4. マッチング機能（場所性嗜好クイズ → おすすめ表示）

### Phase 2b: コンテンツ拡充
1. 施設写真の収集・最適化
2. 場所性スコアのバッチ評価実行
3. 施設ストーリー（物語テキスト）の生成
4. 地域別・テーマ別のキュレーション

### Phase 2c: 全国展開
1. 関西・中部・北海道・九州の施設追加
2. 公開データソース（文化庁DB等）からの自動収集
3. ユーザーからの施設推薦機能

## ワークフロー

```
[仕様策定] → [設計レビュー] → [実装] → [QA+UX] → [修正] → [Sentinel承認] → [デプロイ]
  spec-writer   plan-reviewer   engineer   qa-lead     refinement    sentinel      deploy-manager
                                           ux-lead     coordinator
```

## 次のアクション

1. ユーザーにベンチマーク調査結果を提示し、方向性の承認を得る
2. spec-writerでマッチングサイトの詳細仕様を策定
3. Plan Modeでアーキテクチャ設計
4. 実装開始（Phase 2a MVP）
